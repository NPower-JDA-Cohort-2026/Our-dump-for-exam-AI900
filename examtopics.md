## ANSWER KEY & EXPLANATIONS

---

**A1 — B** · A webchat bot handles common queries automatically, directly reducing the volume of work human agents need to handle.

**A2 — B** · Data is split row-wise so each subset contains all features and labels. Splitting by columns would break the feature-label relationship.

**A3 — TP = 11 | FN = 1033** · True Positives = correctly predicted positive cases. False Negatives = actually positive but predicted negative. Read with Actual on one axis and Predicted on the other.

**A4 — B** · Transparency = AI should be understandable. Enabling model explanation reveals which features drove the prediction and why — opening the black box.

**A5 — No / Yes / No** · Price prediction = regression. Sudden spike = anomaly detection. Diabetes yes/no = classification (supervised, predefined labels).

**A6 — Reliability and safety** · Consistent performance under normal and unexpected conditions, and resistance to manipulation, are core attributes of reliability and safety.

**A7 — Conversational AI / Computer Vision / NLP** · Chatbot = conversational AI. Identifying bottle shapes in images = computer vision. Analyzing text sentiment = NLP.

**A8 — B** · Inclusiveness = AI should benefit everyone including people with disabilities. Fairness is about avoiding bias, not designing for accessibility.

**A9 — Reliability and safety / Accountability / Privacy and security** · Consistent safe operation = reliability and safety. Humans answerable for AI = accountability. Protecting personal data = privacy and security.

**A10 — Reliability and safety** · Resilience to manipulation and safe handling of edge cases are defining characteristics of reliability and safety.

**A11 — C** · Transparency = making AI understandable. Documentation that helps developers understand system behavior directly supports transparency. A = inclusiveness; D = fairness.

**A12 — Regression / Clustering / Classification** · Numeric prediction = regression. Grouping without predefined labels = clustering. Binary yes/no = classification.

**A13 — C** · Inclusiveness is specifically about identifying and removing barriers that could unintentionally exclude people.

**A14 — C, D, F** · The six principles are: Fairness, Reliability & Safety, Privacy & Security, Inclusiveness, Transparency, Accountability. Only C, D, and F appear in the list.

**A15 — Object detection** · Object detection returns what objects are present AND their pixel-coordinate bounding boxes. Classification only labels the whole image.

**A16 — Feature engineering** · Feature engineering creates new, more meaningful inputs from raw data. Feature selection chooses from existing ones.

**A17 — B** · The Face Detect operation returns detected faces plus facial attributes including glasses. Verify checks if two faces are the same person — not needed here.

**A18 — A** · "Explainable" = transparency. Users and regulators must be able to understand how and why a decision was made.

**A19 — Yes / No / No** · Understanding how a model was built = transparency ✅. Claim prioritization relates to fairness/accountability, not privacy. Pricing tiers alone don't map to inclusiveness without context.

**A20 — Fairness / Privacy and security / Transparency** · No discrimination = fairness. Protecting personal data = privacy. Explaining decisions = transparency.

## Quick Reference — ML Task Selection

| Scenario Clue | ML Type |
|---|---|
| Predict a **number** (price, temperature, sales) | **Regression** |
| Predict **yes/no** or a category | **Classification** |
| **Group** data by similarity (no predefined labels) | **Clustering** |
| Detect **unusual patterns or outliers** | **Anomaly Detection** |
| Predict **two outcomes only** | **Binary Classification** |
| Predict **3+ categories** | **Multiclass Classification** |
